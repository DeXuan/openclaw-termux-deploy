# 敏感信息脱敏检查清单

发布公众号文章前必须逐项扫描，全部替换为占位符。**不可跳过**。

## 检查流程

```bash
# 快速扫描（在生成的 HTML 文件上跑）
grep -oE '正则' <文件> | sort -u
```

发现匹配项 → 在 HTML 中替换 → 重新扫描确认 0 残留。

## 14 类清单

| # | 类型 | 正则 | 占位符 | 特殊说明 |
|---|------|------|--------|---------|
| 1 | 内网 IPv4 | `192\.168\.\d{1,3}\.\d{1,3}` | `<局域网IP>` | 192.168.x.x 段 |
| 2 | 公网 IPv4 | 逐值匹配（如 `39\.144\.\d{1,3}\.\d{1,3}`） | `<手机出口IP>` | 蜂窝 CGNAT 出口 |
| 3 | IPv6 | 长十六进制冒号格式 | `<IPv6地址>` | |
| 4 | Tailscale IP | `100\.\d{1,3}\.\d{1,3}\.\d{1,3}` | `<TS_IP>` | 100.x.y.z |
| 5 | API Key | `sk-[a-zA-Z0-9]{20,}` | `<YOUR_API_KEY>` | DeepSeek/OpenAI 格式 |
| 6 | GitHub Token | `gh[op]_[a-zA-Z0-9]{30,}` | `<GITHUB_TOKEN>` | gho_/ghp_ 前缀 |
| 7 | Gateway Token | 40 位十六进制（独立出现、不接其他文本） | `<GATEWAY_TOKEN>` | OpenClaw 的 gateway.auth.token |
| 8 | QQ AppID | `102825839` 等纯数字 9-10 位 | `<YOUR_APP_ID>` | 需上下文确认（非随机数字） |
| 9 | AppSecret | 32 位大小写字母数字混合独立字段 | `<YOUR_APP_SECRET>` | 在 `token`/`secret` 字段上下文 |
| 10 | 密码 | 出现在 `passwd`/`password`/`密码` 上下文 | `<YOUR_PASSWORD>` | 含 `$` `#` 等特殊字符的组合 |
| 11 | 手机号 | `1[3-9]\d{9}` | `<手机号>` | |
| 12 | 设备序列号 | `adb devices` 输出格式（8 位十六进制） | `<设备SN>` | |
| 13 | GitHub 用户名 | `DeXuan` 等具体 ID | `<your-username>` | repo URL 里出现时替换 |
| 14 | 设备具体型号 | `23013RK75C` 等 | 保留型号名如 `Redmi K60 Pro`，去掉内部代号 | 型号名可保留（技术文章环境说明），内部代号必须去 |

## 脱敏原则

1. **宁可多替换，不可漏一个**
2. IP 类：`127.0.0.1` 不替换（localhost，无害）；其余全换
3. 占位符选语义化的（读者一看就懂），不要用 `xxx` 或 `***`
4. 同一个值多次出现 → 统一替换为同一个占位符
5. 替换完跑 `grep` 二次确认（见上方命令）
